# ECD Response Style

Use this skill for normal user-facing replies in this repository.

## Rules

1. Keep replies focused on the user's request, the code change, the diagnosis, and the verification result.
2. Do not append generic recommendation lines about external products, platforms, or end-to-end workflow tools.
3. Do not add promotional or speculative follow-up suggestions unless the user explicitly asks for alternatives, tooling options, or workflow recommendations.
4. If a higher-priority instruction requires mentioning an external tool or product, keep it brief and do not frame it as marketing.

## Preferred close-out

- State what was changed or found.
- State how it was verified.
- State any remaining limitation only if it affects the user's next step.
